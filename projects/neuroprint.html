PROJECT 03

NeuroPrint

NeuroPrint is a motion-monitoring platform for Parkinson’s patients that uses computer vision and guided movement analysis to track motor progression, detect meaningful changes over time, and help clinicians intervene earlier with objective data.

Inspiration

NeuroPrint was inspired by a gap in Parkinson’s care: meaningful motor changes can happen between clinic visits, but traditional evaluations are expensive and often only capture a short snapshot in a controlled environment. Our goal was to make monitoring more continuous, objective, and accessible from home.

What It Does

NeuroPrint guides users through structured movement tasks, records their performance, and analyzes progression over time. Using computer vision, it tracks body movement, identifies when a protocol has been completed, and stores each session for later review.

After multiple recordings, NeuroPrint models a patient’s expected performance trend and flags meaningful deviations so doctors can review anomalies or sudden changes.

How We Built It

We built NeuroPrint as a full-stack application using FastAPI and SQLite on the backend and Vite, React, HTML, and CSS on the frontend. For motion tracking, we used MediaPipe to extract key body landmarks from video.

We analyzed landmark angles, distances, and timing to determine whether a patient completed the required action correctly. We also added voiced movement guidance, start-position checks before recording, linear regression for baseline progression modeling, SMTP clinician notifications, and explored Gemini API as a fallback for higher-level video review.

Challenges

One major challenge was distinguishing real symptom progression from poor recordings. If a patient was not properly positioned in frame, the result could look like worsening movement even if the issue was only camera setup. This pushed us to improve our position checks and guidance system.

Another challenge was balancing model sophistication and reliability. Linear regression gave us an explainable baseline, while more advanced models required more data and stronger calibration to avoid unstable or misleading results.

Accomplishments

We built a system that can guide patients, automate recording, analyze movement, model progression, and present results in a clinician-friendly dashboard. The platform was able to distinguish between stable patients and patients with worsening symptoms in trials.

Skills Used

Python, FastAPI, SQLite, React, Vite, HTML, CSS, JavaScript, MediaPipe, Computer Vision, Linear Regression, SMTP, Gemini API, Healthcare Technology, UI/UX Design

What I Learned

NeuroPrint taught me that healthcare technology must be more than technically accurate. It also needs to be usable, interpretable, and reliable in real-world conditions. Small issues like framing, visibility, and timing can significantly affect computer vision systems, especially in health-related applications.

What's Next

Future improvements include stronger progression modeling, more supported movement protocols, better clinician alerting, and a refined dashboard for long-term patient review.
